# Guide for AI: Niche4Niche Studio (theme + optional full page)

## What Niche4Niche is

Niche4Niche is a music identity platform. Users organize **sections** of songs (Spotify, YouTube, SoundCloud links + cover art). The app can render a live page from that library data and a **theme stylesheet**.

## Recommended workflow: AI writes **CSS only** (default)

1. User adds sections and songs in **Studio → Data**.
2. User opens **Studio → Code** and copies the **Theme (CSS only)** prompt from the AI prompt panel.
3. The model returns **one** ` ```css ` block. User pastes into the **CSS** field. **HTML** stays generated by the app (read-only unless **Custom HTML** is on).
4. New sections and songs **automatically** use the same theme—class names on the markup are stable.

**Rules for theme-only output:** plain CSS only. Do **not** change or fabricate stream URLs or cover `src` values (the app owns the HTML). Do not add platform **iframes/embeds** unless the user explicitly asks. Style layout, color, type, and surfaces only.

The exact markup tree is documented in `DATA_PAGE_MARKUP_REFERENCE` inside `lib/profile-template.ts`.

## Optional workflow: full custom HTML + CSS

If the user turns on **Custom HTML** in Studio, they may paste **both** HTML and CSS. Then:

- Output **two** blocks: ` ```html ` (body fragment only) then ` ```css `.
- **Embeds:** If generating HTML, use the **exact** `href` and `img src` from the user’s library list—do not swap in different artwork or links. Prefer link+image **cards** over iframes unless the user requests embed players.
- No `<script>`, no React/Tailwind, no document wrappers—same as before.

## Where this code goes

**Studio → Code** (`/`):

- **CSS** — always used as the page theme (or with custom HTML).
- **HTML** — generated from library by default; editable only with **Custom HTML** enabled.

## Markup the app generates (theme mode)

The live HTML includes:

- **Profile header:** `.hero` with `.name`, `.handle`, optional `.bio`
- **Sections:** `.sections` → `.section` → `.section-head` (`h2`, optional `.mood`) → `.song-grid`
- **Songs:** `a.song-card` with `.song-art img`, `.song-info`, `.song-title`, `.song-artist`, `.platform-dot`
- **Empty section:** `.empty` paragraph
- **Footer:** `.site-footer`

## Output formats

### Theme mode (default)

- **Exactly one** fenced ` ```css ` block. No HTML.

### Custom HTML mode

1. **HTML** — body inner markup only (no `<html>`, `<head>`, `<body>`, `<style>`).
2. **CSS** — full stylesheet for that HTML.

## Constraints (both modes)

- No `<script>` unless the user explicitly asks.
- No frameworks — plain HTML and/or CSS only.
- HTTPS for fonts and external assets.
- Valid, balanced HTML when HTML is produced.
- No inline event handlers.

## Quality

Intentional typography and spacing; at least one mobile `@media` rule; readable class names when authoring HTML.

## AI prompts in the repo

- **Theme:** `buildStudioAiThemePrompt()` in `lib/ai-studio-prompt.ts`
- **Full page:** `buildStudioAiFullPagePrompt()` in the same file

In-app: **Studio → Code → AI prompt → Copy theme prompt (CSS only)** or **Copy full HTML + CSS prompt**.

### Manual short prompt (theme only)

```
Niche4Niche: output ONE Markdown fenced block labeled css, nothing else. Style this tree: .page .hero .name .handle .bio .sections .section .section-head .mood .song-grid .song-card .song-art img .song-info .song-title .song-artist .platform-dot .empty .site-footer. Plain CSS, https fonts ok, one @media for mobile. Do not output HTML. Do not fake cover or stream URLs.

Creative direction: [VIBE]
```
